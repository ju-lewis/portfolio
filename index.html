03. Machine learning projects

Handwritten Digit Classifier:

This is a simple no dependency pure JavaScript digit classifier I wrote as a small two day side project. The neural network is a simple feed-forward network with no hidden layers and a softmax output. For training it uses cross-entropy loss and simple gradient descent, with Xavier initialised weights.
The maximum accuracy isn't fantastic as the model is incredibly simplistic, but it was a fun experiment to program a browser-native neural network.
When you click train it requests the MNIST dataset from the server and begins training on all digits from 0 to 9. You can draw in the white canvas and press 'Predict' to interact with the model!

The interactive neural network demo is only available on desktop.

04. Embedded development projects

Point-Track

Point-Track is an ongoing software development and electrical engineering project I'm working on to automate the scoring and management of yacht time trial races. The project involves two main parts: a microprocessor and GNSS receiver which are kept on a boat, and a web application to process and display results, managed by the yacht race organiser.

Goal:

Create a small and affordable GNSS positioning unit that can record sub-metre accuracy position, velocity, and heading data that can be wirelessly transferred to a central server after a race. This positioning data needs to be compared against a pre-defined selection of points with corresponding 'arrival times' to determine the overall accuracy of the boat based on how close it was to the pre-defined course.

  1. For the positioning unit on the boats, I elected to use an ESP-32 microcontroller for its native wireless capabilities and built-in SPIFFS file system support. This microcontroller was responsibile for driving both the user interface (comprised of 4 buttons and a 16x2 LCD display) and reading data from the GNSS receiver module (for which I chose a U-Blox ZED F9P for its raw output ability for post-processing kinematic correction, enabling low-cost high-accuracy poositioning).
  2. For the web application, I built the entire system with minimalism and simplicity of user interaction in mind so I utilised Rust for its speed and runtime safety guarantees, with my API design philosophy being entirely based on server rendered hypertext - doing away with an intermediate data representation like JSON.
